Everything Different

SIKKA rethinks every layer of a digital currency — from the ledger structure to the cryptography to the network transport.

Zero Fees — Forever
When you send SIKKA, the exact amount arrives in full. There are no gas fees, no priority fees, no miner tips, no protocol cuts. Zero. The recipient receives exactly what you sent — always.

Blockless DAG Architecture
SIKKA has no blocks and no chain. Every transaction is a vertex in a directed acyclic graph. Transactions are confirmed in parallel — not serialized into blocks — delivering true horizontal scalability.
∞
Parallel TXs
0

Exactly Two Parents
Every non-genesis transaction references exactly two prior transactions as parents. This invariant is what makes SIKKA a true DAG — not a list, not a tree.
Exact Balance Model
Outputs must sum exactly to inputs. No change outputs, no rounding. The protocol enforces arithmetic precision at the consensus layer — not the wallet layer.
Parallel Scaling
As more nodes join, throughput increases. No global sequencer. No leader. Every honest node processes transactions in parallel with full deterministic convergence.
Spam Resistance

Per-Transaction Proof of Work

SIKKA requires every transaction to include a SHA3-256 nonce solution. Difficulty scales dynamically with network load — spam gets expensive, normal use stays free.

Security

Post-Quantum by Design

No classical elliptic curve fallbacks. Every key, every signature, every hash uses algorithms hardened against quantum computers.

ML-DSA-87
NIST FIPS 204 Post-Quantum
SIKKA uses Module-Lattice-Based Digital Signature Algorithm at the highest security level (87 = NIST Level 5). This is the strongest post-quantum signature standard published by NIST. There are no classical ECDSA or Ed25519 fallbacks — every signature in SIKKA is quantum-resistant by default.
