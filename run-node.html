Installation

One command to rule them all

No dependencies. No compilation. No configuration files. Copy, paste, done.

bash — docker run 
docker run -d --restart unless-stopped \
  -p 64552:64552 \
  -v sikka-data:/home/sikka/data \
  ghcr.io/sikkalabs/sikka:latest
-d
Detached mode — runs in background
--restart
Auto-restarts unless you stop it manually
-v
Persistent volume for DAG data
What's Included

Everything in 17 MB

The SIKKA binary is a self-contained node. No runtime dependencies, no sidecar processes, no package managers.

Full Web Wallet
Create and manage ML-DSA-87 keypairs, send and receive SIKKA, and build multisig transactions — all from your browser, locally.
Open wallet
Built-in Explorer
Real-time DAG visualization powered by D3. Browse every transaction, inspect parent links, monitor tips, and track network health live.
Full DAG Engine
Complete consensus implementation. Validates transactions, resolves conflicts, syncs with peers, and maintains the full SIKKA ledger state.
Managed Tor
Tor is bundled inside the binary. Your node automatically generates a .onion address and routes all traffic over Tor. Zero configuration required.
HTTP REST API
Full JSON API for submitting transactions, querying DAG state, fetching transaction details, and monitoring network status on port 64552.
Peer Federation
Gossips with other SIKKA nodes to stay in sync. Discovers peers over the SIKKA network automatically — no boot node list to configure.
Setup Guide

Up in 3 steps

1

Install Docker

Docker Desktop (Mac/Windows) or Docker Engine (Linux). Any recent version works. That's the only prerequisite.

Get Docker
2

Run the command

Copy the Docker command above and paste it into your terminal. The image pulls automatically from the GitHub container registry.

docker run -d --restart unless-stopped -p 64552:64552 …
3

Open the wallet

Navigate to http://localhost:64552 in your browser. The web wallet and DAG explorer are live. Your node is syncing and your .onion address is active.
